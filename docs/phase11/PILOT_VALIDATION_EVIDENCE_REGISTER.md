## How to use this register

Each evidence item receives an ID:

```text
P11-<STAGE>-<TYPE>-###
```

Examples:

```text
P11-C-ONBOARD-001
P11-D-TRUST-001
P11-G-DEVICE-001
P11-H-PRICE-001
```

Every result must identify:

- sample/cohort count without exposing identity;
- exact approved pilot scope version;
- device/network context where relevant;
- observation versus interpretation;
- defect/assumption/request classification;
- linked issue/commit when engineering action follows;
- whether revalidation passed;
- limitations and contradictory evidence.

Do not pre-fill results with synthetic numbers.
