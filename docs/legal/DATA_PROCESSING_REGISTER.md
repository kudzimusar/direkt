# DIREKT Data Processing Register

Populate and legally review before pilot.

| Activity | Data subjects/data | Purpose | Basis/consent | Recipients | Retention | Controls |
|---|---|---|---|---|---|---|
| Account | contact, session/device | access/security | to be reviewed | auth/SMS vendors | bounded | encryption/rate limits |
| Provider verification | identity, certificates, private location | trust checks | to be reviewed | reviewers/approved sources | policy | private storage/audit |
| Discovery | public provider/service area, customer search area | match providers | service | map vendor limited | minimal | precision controls |
| Enquiry | customer/provider contact, area, description | interaction | service/consent | parties, notification vendors | policy | sharing consent |
| Reviews/complaints | interaction/opinion/evidence | accountability/safety | to be reviewed | trust staff/parties limited | policy | moderation/access |
| Subscription | billing/contact/transactions | commercial | contract/legal | payment/accounting | statutory | ledger/reconciliation |
| Analytics | app events/coarse context | reliability/product | to be reviewed | analytics processor | aggregate early | no evidence/private coords |

Each new processing activity requires owner, classification, vendor and deletion implementation.
