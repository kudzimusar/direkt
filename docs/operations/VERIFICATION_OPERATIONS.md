# DIREKT Verification Operations

## Queue design

Queues by check, category, risk, age/SLA, geography and assignment. Automatic ordering must not hide urgent safety/fraud.

## Standard operating procedure

- confirm case/requirement version;
- inspect completeness and automated checks;
- compare identity/provider fields;
- use checklist;
- select decision and reason;
- preview public/provider message;
- submit;
- escalate uncertainty/conflict.

## Quality assurance

Sample approvals/rejections, disagreement review, calibration sessions, issuing-body updates, fraud pattern review and agent performance that does not reward approvals.

## Capacity

Measure volume, handling time, rework, field need, backlog and cost per check. Stop onboarding if backlog threatens safe quality.

## Access

Privileged, audited, time-limited evidence access; no downloading to personal devices.

## Changes

Requirement/checklist/template changes are versioned and trained before effective date.
