# DIREKT Data Retention

## Principles

- retain only for stated purpose;
- shortest defensible period;
- distinguish active, historical, legal hold and deletion;
- evidence has stricter access than public profile;
- deletion does not rewrite required audit/financial history;
- legal review finalizes periods.

## Data classes

| Class | Baseline approach |
|---|---|
| Public provider profile | while active plus controlled history |
| Identity/qualification evidence | active validity + appeal/fraud/legal period |
| Rejected abandoned upload | short cleanup period unless fraud/appeal |
| Private location/field evidence | active need + controlled audit period |
| Enquiries | support/review eligibility period |
| Reviews | while published + moderation history |
| Complaints/incidents | severity/legal schedule |
| Financial records | statutory/accounting schedule |
| Security logs | bounded operational schedule |
| Analytics | aggregate/de-identify early |

## Deletion workflow

Authenticate request, identify obligations, restrict processing, delete/anonymize eligible data, preserve required audit with minimization, notify completion and propagate to vendors/backups according to policy.

## Holds

Legal/safety/fraud hold requires authority, scope, reason, start, review and release.
