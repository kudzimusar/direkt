# DIREKT Acceptance Tests

## AT-001 Customer discovery

Given a published provider serving the selected area, customer can find the provider in list and map, inspect current check details and submit an enquiry without granting location permission.

## AT-002 Provider verification

Given a provider draft and category requirements, provider can submit evidence; reviewer can request correction; provider can resubmit; approval creates only the scoped public claim.

## AT-003 Private location

Given a mobile provider with private verified base, public API/app shows service area and “location checked” summary but never exact private point/address.

## AT-004 Expiry

When a mandatory qualification expires, public claim and verification summary degrade automatically, filters update and provider receives action.

## AT-005 Review integrity

Only a customer with eligible tracked interaction can submit standard review. Provider cannot review itself.

## AT-006 Commercial separation

A subscription activation, failure or upgrade never creates, extends or restores a verification check.

## AT-007 Operations audit

Reviewer evidence access and decisions are visible in immutable audit to authorized auditor.

## AT-008 Offline provider draft

Provider loses connectivity during onboarding/upload; entered data remains recoverable and no false “submitted/approved” state appears.

## AT-009 Suspension

Trust supervisor can suspend according to policy; provider leaves discovery promptly; appeal/history remains.

## AT-010 Accessibility

Critical customer and provider flows are usable with TalkBack, large font and list alternative to map.
