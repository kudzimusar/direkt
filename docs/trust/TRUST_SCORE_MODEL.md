# DIREKT Trust Score Model

## Status

A public composite trust score is **not approved for MVP launch**. The platform will first show individual checks, response and review facts. This document defines safeguards if a score is later tested.

## Goals

- improve ranking relevance;
- expose current, explainable signals;
- avoid pay-to-win;
- avoid hidden discrimination;
- degrade stale evidence;
- permit correction and appeal.

## Candidate inputs

- mandatory check completeness/currentness;
- profile/location freshness;
- response rate/time with minimum sample;
- verified-interaction review quality with Bayesian/minimum-sample treatment;
- substantiated complaint/enforcement history;
- re-verification consistency;
- no-show/cancellation facts if reliably captured.

## Excluded inputs

- payment tier;
- protected characteristics;
- proxies likely to create unlawful discrimination;
- raw neighbourhood wealth;
- popularity alone;
- unverified social-media claims;
- private complaint allegation before due process;
- device price/type.

## Explainability

Any score displayed must show major contributing categories and improvement/correction routes. Customers must understand it is an indicator, not a safety guarantee.

## Governance

- documented formula/version;
- offline evaluation;
- bias and gaming review;
- threshold/minimum sample;
- monitoring for category/geography effects;
- human appeal;
- rollback;
- owner/legal approval before public use.

## MVP alternative

Use explicit filters and ranking features with clear facts: current required checks, service match, distance, availability, response and eligible-review summary.
