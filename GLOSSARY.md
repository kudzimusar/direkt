# DIREKT Glossary

| Term | Controlled meaning |
|---|---|
| Customer | A person or organization searching for or interacting with a provider |
| Provider | A person, sole trader, business or organization offering services |
| Provider member | A user authorized to manage a provider profile |
| Service category | Controlled classification such as plumbing or electrical services |
| Service area | Geographic area in which a provider states they operate; not proof of presence |
| Public location | Coarsened or consented location shown to customers |
| Private evidence location | Precise coordinate/address retained only for verification and authorized operations |
| Fixed-premises provider | Provider serving customers at a stable business location |
| Mobile provider | Provider travelling to customers without a public storefront |
| Hybrid provider | Provider with premises and mobile service |
| Verification check | One scoped examination, such as phone, identity, qualification or premises |
| Verification case | Operational record grouping a check, evidence, reviewer actions and outcome |
| Evidence | Document, image, structured response, external confirmation or field record used in a check |
| Claim | Publicly displayable, backend-derived statement about an approved check |
| Badge | Visual indicator representing a specific claim; never a universal safety guarantee |
| Verification level | Summary of completed check groups; individual check details remain visible |
| Field agent | Authorized person conducting physical or in-person verification |
| Good standing | Current status based on non-expired evidence and absence of disqualifying enforcement |
| Trust score | Explainable ranking signal derived from approved data; not a guarantee |
| Enquiry | Platform-tracked customer request to a provider |
| Interaction | Platform record proving that customer/provider contact or service flow occurred |
| Verified review | Review eligible because of a platform-tracked interaction; it does not mean the opinion is endorsed |
| Suspension | Temporary restriction pending review or remediation |
| Revocation | Withdrawal of an approved check or claim |
| Delisting | Removal from public discovery while records are retained according to policy |
| Operations portal | Internal web application for verification, support, moderation and administration |
| Pages | Public static GitHub Pages documentation/prototype site; not the DIREKT backend |
| MVP | Minimum product completing the real provider-verification-to-customer-interaction loop |
